## Лабораторная работа №2

### Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели

---

#### *(Вариант 5)*

### Структура сервисов Microsoft Azure
#### Cloud Services

#### Назначение:
Основная категория для вычислительных, платформенных и прикладных сервисов, обеспечивающих запуск приложений, обработку данных и выполнение вычислений без прямого управления физической инфраструктурой.

---

### Семейства сервисов

#### Application Services
Сервисы для развертывания, выполнения и сопровождения приложений.

#### Azure Batch
Сервис пакетной обработки и выполнения массовых вычислительных задач.

#### Классификация:
PaaS, так как пользователь управляет заданиями и логикой выполнения, но не управляет виртуальными машинами, сетью и операционной системой.

#### Тип оплаты:
Job Compute Hours - оплата за фактическое время выполнения вычислительных заданий

#### Аналог в AWS:
AWS Batch

---

#### Azure SignalR Service
Управляемый сервис обмена сообщениями в реальном времени для веб- и мобильных приложений.

#### Классификация:
PaaS, поскольку инфраструктура и масштабирование скрыты от пользователя, требуется только конфигурация и использование API.

#### Тип оплаты:
Message Units — оплата за количество переданных сообщений

#### Аналог в AWS:
Amazon API Gateway WebSocket / Amazon AppSync

---

### Analytics Services
Сервисы для аналитики, обработки и визуализации данных.

#### Azure Analysis Services
Управляемая аналитическая платформа для построения табличных моделей и аналитических решений.

#### Классификация:
PaaS, так как сервис предоставляет готовую аналитическую платформу без необходимости управления инфраструктурой.

#### Тип оплаты:
Analytics Capacity - оплата за зарезервированные вычислительные мощности

#### Аналог в AWS:
Amazon Redshift / Amazon Athena (частично)

--- 

#### Azure Databricks
Платформа распределённой аналитики и обработки больших данных на базе Apache Spark.

#### Классификация:
PaaS, так как пользователь работает с кластерами и заданиями, но не управляет базовой инфраструктурой.

#### Тип оплаты:
Cluster Usage - оплата за время работы кластеров и вычислительные ресурсы

#### Аналог в AWS:
Amazon EMR

---

### Artificial Intelligence / Machine Learning
#### Azure Machine Learning
Платформа для создания, обучения, развертывания и сопровождения моделей машинного обучения.

#### Классификация:
PaaS, так как предоставляется управляемая среда и инструменты для ML-задач, а не готовые модели.

#### Тип оплаты:
Training / Inference - оплата за обучение и инференс моделей

#### Аналог в AWS:
Amazon SageMaker

---

### Management & Monitoring
#### Azure Monitor
Сервис мониторинга, сбора логов и телеметрии для ресурсов Azure.

#### Классификация:
SaaS, поскольку предоставляется готовый инструмент мониторинга без необходимости настройки инфраструктуры.

#### Тип оплаты:
Metrics / Logs - оплата за объём собранных метрик и логов

#### Аналог в AWS:
Amazon CloudWatch

---

### Database Services
*(Аналог в AWS: Data / Storage)*

#### Назначение:
Сервисы для хранения и обработки структурированных данных.

#### Azure SQL Database / Azure Database for PostgreSQL / MySQL
Полностью управляемые реляционные базы данных.

#### Классификация:
PaaS, так как пользователь работает только с данными и запросами, управление СУБД и инфраструктурой осуществляется провайдером.

#### Тип оплаты:
DB Usage - оплата за вычислительные и хранилищные ресурсы базы данных

#### Аналог в AWS:
Amazon RDS

---

### Storage Services
#### Azure Storage
Облачные хранилища данных (объектное и блочное).

#### Классификация:
IaaS, так как предоставляется инфраструктурное хранилище данных, а пользователь управляет структурой хранения и доступом.

#### Тип оплаты:
Storage Usage - оплата за объём данных и операции

#### Аналог в AWS:
Amazon S3 / Amazon EBS

---

### Backup & Recovery
#### Azure Backup / Recovery Services
Централизованные сервисы резервного копирования и восстановления данных.

#### Классификация:
SaaS, так как пользователь использует готовое решение для резервного копирования без управления инфраструктурой.

#### Тип оплаты:
Backup Storage / Restore — оплата за хранение резервных копий и операции восстановления

#### Аналог в AWS:
AWS Backup

---

### Выводы

В ходе выполнения лабораторной работы была сформирована кросс-провайдерная сервисная модель Microsoft Azure с сохранением логической структуры.

Большинство сервисов Azure относятся к модели PaaS, что отражает ориентацию платформы на предоставление управляемых сервисов и снижение ответственности пользователя за инфраструктуру.
AWS, в свою очередь, предоставляет более широкий выбор IaaS-сервисов, обеспечивая больший уровень контроля над вычислительными и сетевыми ресурсами.

Модель оплаты в обоих облаках преимущественно основана на:

потреблении ресурсов (consumption-based),

либо резервированных мощностях (capacity-based),
а оплата строго за время использования встречается реже.

Сервисы Azure тесно интегрированы между собой и образуют единую экосистему, в то время как AWS делает больший акцент на модульность и специализированные сервисы.
