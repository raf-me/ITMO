## Лабораторная работа №1

Тема: Знакомство с моделями IaaS, PaaS и SaaS на примере Amazon Web Services. Создание сервисной модели.

---

### Цель работы:

Целью лабораторной работы является изучение облачных сервисов Amazon Web Services (AWS), а также понимание уровней абстракции над инфраструктурой в облаке: IaaS, PaaS и SaaS.
Дополнительной задачей является формирование навыков классификации потребления облачных ресурсов и построения сервисной модели, удобной для аналитики и бизнес-анализа.

---

### Исходные данные:

В качестве исходных данных был предоставлен файл биллинга AWS в формате CSV, предварительно обработанный и приведённый к виду, близкому к SQL-параметрам.
В столбцах Product Code и Usage Type содержатся технические идентификаторы сервисов и типов их использования.
Символ % в значениях Usage Type означает, что до или после него может находиться произвольная последовательность символов.

Для выполнения работы использовался один файл варианта — “Mapping Rules AWS team 4.csv”.

---

### Задание:

Необходимо:

1. Импортировать CSV-файл в табличный редактор (Excel).

2. На основе документации AWS сопоставить биллинговые данные с конкретными сервисами.

3. Заполнить первые пять столбцов таблицы:

- IT Tower

- Service Family

- Service Type

- Service Sub Type

- Service Usage Type

4. Построить иерархию сервисов от общего уровня к частному.

5. Сохранить заполненный файл и загрузить его в соответствующую папку Google Drive.

---

### Краткая теория

IaaS (Infrastructure as a Service) - модель, при которой пользователь арендует вычислительные ресурсы (виртуальные машины, сети, хранилища), самостоятельно управляя операционными системами и конфигурацией.

PaaS (Platform as a Service) - модель, при которой провайдер предоставляет готовую платформу, снимая с пользователя ответственность за управление инфраструктурой и ОС.

SaaS (Software as a Service) -- модель, при которой пользователю предоставляется готовое программное обеспечение, доступное через интернет.

### Ход выполнения работы

После импорта файла в Excel были включены фильтры по столбцам Product Code и Usage Type.
Далее все строки были сгруппированы по уникальным сервисам AWS. Несмотря на большое количество строк в биллинге, фактически в файле присутствует ограниченное число сервисов, каждый из которых имеет несколько вариантов потребления.

Для каждого сервиса на основе официальной документации AWS были определены:

- уровень абстракции (IaaS / PaaS / SaaS),

- принадлежность к IT Tower,

- семейство сервисов,

- тип и подтип сервиса,

- человекочитаемый способ потребления ресурса.

Заполнение таблицы производилось блочно, то есть значения копировались сразу на все строки с одинаковым сервисом и типом использования.

### Результаты классификации сервисов (team 4)
### IaaS

Виртуальные вычислительные ресурсы - Amazon EC2
Сервис предоставляет виртуальные машины с полным контролем со стороны пользователя. В биллинге отражено использование стандартных инстансов и выделенных хостов (Dedicated Host), что соответствует инфраструктурной модели потребления.

Виртуальные рабочие места - Amazon WorkSpaces
Облачные виртуальные десктопы, предоставляющие пользователю инфраструктуру для удалённой работы без необходимости развёртывания собственной VDI-системы.

---

### PaaS

Бессерверные вычисления - AWS Lambda
Платформа для выполнения кода без управления серверами. Оплата осуществляется за время выполнения функций, что характерно для модели Platform as a Service.

Обработка больших данных - Amazon EMR (Elastic MapReduce)
Полностью управляемая платформа для обработки данных на базе Hadoop и Spark. Пользователь работает с кластерами и заданиями, не управляя инфраструктурой напрямую.

---

### SaaS

Мониторинг и логирование - Amazon CloudWatch
Сервис предоставляет готовые инструменты для сбора метрик, логов и мониторинга ресурсов AWS. Пользователь использует функциональность без управления инфраструктурой.

Сервисы безопасности - Amazon Inspector, Amazon Detective, Amazon Macie
Управляемые решения для анализа уязвимостей, расследования инцидентов и обнаружения чувствительных данных. Используются как готовые сервисы безопасности.

Бизнес-приложения - Amazon WorkMail, Alexa for Business (A4B)
Готовые облачные сервисы для корпоративной почты и голосовых помощников.

Аналитика и взаимодействие с пользователями - Amazon Pinpoint, mobileanalytics
Сервисы для анализа пользовательской активности и взаимодействия с клиентами.

Управление затратами - AWS Budgets
SaaS-сервис для контроля и мониторинга расходов в AWS.

Тестирование приложений - AWS Device Farm
Облачный сервис для тестирования мобильных и веб-приложений на реальных устройствах.
